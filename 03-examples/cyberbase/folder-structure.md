# Cyberbase Folder Structure

> [!important] This will change over time since folders are a strict hierarchy and my goals and system capabilities will surely change over time.

Uses [SEACOW(r)](../../00-start-here/quick-start.md) — my meta-framework for organizing knowledge platforms.

(Uses emojis to add a level of sorting and visuals — see related cyberbase notes on file naming and file order.)

```
_attachments       — (SYSTEM)
_excalidraw        — (SYSTEM)
✅ TASKS           — (SYSTEM)
⬇️ Clipping        — (CAPTURE)
⬇️ INBOX, DROPZONE — (CAPTURE)

(OUTPUT using Johnny Decimal-like naming system for sorting)
📁 01 - Projects             — (OUTPUT)
📁 02 - CyberNews            — (OUTPUT)
📁 03 - Curations, Stacks    — (OUTPUT)
📁 04 - Cyber & Digital Trust— (OUTPUT)
📁 05 - Organizational Cyber — (OUTPUT)
📁 06 - Learning, Notes      — (OUTPUT)
📅 Changelog                 — (OUTPUT)
🕸️ UNSTRUCTURED              — (CAPTURE, WORK)
🕸️ Daily Notes               — (CAPTURE, RELATION)
```

## How this maps to SEACOW(r)

| Folder | Activities | Notes |
|---|---|---|
| `_attachments`, `_excalidraw`, `✅ TASKS` | SYSTEM | The platform's own affordances; not content. |
| `⬇️ Clipping`, `⬇️ INBOX, DROPZONE` | CAPTURE | Web clipper output and inbox-style raw drops. |
| `📁 01..06` | OUTPUT | Audience-facing structure. Johnny Decimal-style numbering forces stable sort and easy navigation. |
| `🕸️ UNSTRUCTURED` | CAPTURE + WORK | Things that aren't ready for OUTPUT yet — work-in-progress synthesis. |
| `🕸️ Daily Notes` | CAPTURE + RELATION | Daily journals are both raw input *and* a connecting thread across notes. |
