# Case studies

Community-submitted instantiations of SEACOW(r). Empty for v0.1.0 — submit yours via PR.

## What to include in a case study

1. **Context** — what platform, who's the entity, what's the goal?
2. **Constraints** — what did the system afford / forbid?
3. **Mapping** — for each SEACOW(r) component, what structural choice did you make?
4. **Trade-offs** — what didn't fit cleanly?
5. **What you'd change** — what would you do differently next time?

## Suggested file name

`<your-context>.md`, kebab-case. Examples: `personal-research-vault.md`, `compliance-team-fileshare.md`, `student-homework-folder.md`.
