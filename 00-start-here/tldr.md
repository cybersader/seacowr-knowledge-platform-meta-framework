# TL;DR — SEACOW(r) in 60 seconds

SEACOW(r) is a **lens** for designing knowledge organization. Not a folder
structure. Not a template. A set of questions you answer for **your** context.

| Letter | Question |
|---|---|
| **S** — System | What platform are we in? What does it afford (folders, tags, links, properties)? What does it constrain? |
| **E** — Entity | Who accesses this? Just me? A team? The public? AI agents? |
| **A** — Activities | The three things that happen to information: |
| ↳ **C** — Capture | Where does information *enter* the system? |
| ↳ **O** — Output | Where does it *exit* to a consumer? |
| ↳ **W** — Work | Where does it get *processed*? |
| **r** — relation | How do things *connect* across the structure? (Often the graph layer — tags, links, relations.) |

**The core insight:** different organization frameworks (PARA, Zettelkasten,
Johnny Decimal, LATCH, ARCO) prescribe different folder shapes. SEACOW(r) is
upstream of all of them — it asks what the folders are *for*, then lets you
pick (or invent) a shape that serves your purpose.
