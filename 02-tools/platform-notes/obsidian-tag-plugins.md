# Obsidian tag & folder plugin ecosystem

Plugins that **use or affect structure** in Obsidian — relevant when applying
SEACOW(r) because they extend (or constrain) what Capture / Work / Output /
Relation can do via the System.

## Direct tag-folder bridging / structure manipulation

- **MyThesaurus.**
- **TagFolder** — [vrtmrz/obsidian-tagfolder](https://github.com/vrtmrz/obsidian-tagfolder) — view tags as a folder-style hierarchy in the sidebar.
- **Tag Wrangler** — [Obsidian Stats: Tag Wrangler — Rename](https://www.obsidianstats.com/plugins/tag-wrangler) — rename / merge / refactor tags.
  - **Tag Pages** (folded in from cyberbase's separate "Tag Pages" note): Tag Wrangler enables tag-as-page workflows where a tag is also a note.
- **Folder by tags distributor** — moves notes to folders based on tags. *Doesn't handle unstructured notes other than sending them to root.*
- **Folder Notes.**
- **AutoMover** — [al0cam/AutoMover](https://github.com/al0cam/AutoMover) — moves files with specified names into the same folder.
- **Auto Note Mover** — [farux/obsidian-auto-note-mover](https://github.com/farux/obsidian-auto-note-mover) — auto-moves notes based on rules.
