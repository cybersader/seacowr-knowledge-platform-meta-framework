# Obsidian tag bugs & issues

A short reference of known Obsidian rendering bugs that affect SEACOW(r) tag
schemes. When designing prefix conventions, factor these in.

## Tags beginning with underscores

> [!info] Underscores still have issues when used to start a tag (at least in live preview mode).

- [Tag beginning with an underscore mess up the live preview render — Obsidian Forum](https://forum.obsidian.md/t/tag-beginning-with-an-underscore-mess-up-the-live-preview-render/41470)
- [Tag starting with underscore makes the rest of the line italic — Obsidian Forum](https://forum.obsidian.md/t/tag-starting-with-underscore-makes-the-rest-of-the-line-italic/79705/2)

If you adopt the cyberbase tag prefix convention (`#_publicTaxonomy/...` for
OUTPUT tags), watch out for these rendering bugs in live preview. Reading
view is unaffected.
