## How to read these

Each page describes the other framework on its own terms first, then maps it to
SEACOW(r) components. SEACOW(r) is **not a replacement** for any of these —
it's a meta-layer that helps you choose between them, or compose them.

## Pull-request welcome

Got another framework? Open an issue or submit a PR with a stub. Format:

1. One-paragraph summary of the other framework (its own vocabulary).
2. Mapping table: their concept → SEACOW(r) component(s).
3. Where they overlap, where they diverge.
4. When you'd pick which.
