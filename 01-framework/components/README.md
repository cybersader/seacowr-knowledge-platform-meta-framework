# SEACOW(r) components

One page per component. Each page follows the same shape:

1. **Question** — the diagnostic question the component asks.
2. **Examples** — across multiple platforms (Obsidian, file share, code repo, Notion, physical).
3. **Analysis** — what design decisions the component drives.
4. **Anti-patterns** — common mistakes when prescribing rather than analyzing.
