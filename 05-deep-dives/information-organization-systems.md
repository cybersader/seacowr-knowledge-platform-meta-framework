# Information Organization Systems

See: [meta-framework](../01-framework/meta-framework.md).

## Components of organization

> [!summary] These are ways to mentally model how and why to organize information.

### Goals of organization

1. **Information Retrieval** — finding the thing you need.
2. **Knowledge Work** — working *with* information from within the system itself (libraries aren't designed for this).
3. **Action** — the goal of some organizational systems (planners, todo lists, GTD) is helping you act / reach goals.
4. **Knowledge Capture / Inputs** — some systems just need to be easy to put stuff into boxes.

### Things we can organize by

- **STIR**:
  - **S**pace
  - **T**ime
  - **I**mportance
  - **R**elatedness
- **LATCH**:
  - **L**ocation
  - **A**lphabet
  - **T**ime
  - **C**ategory
  - **H**ierarchy (the original Wurman version called this "Continuum")
- **Ways to organize:**
  - By the **process** (input, organizing, output) → maps to SEACOW(r) Activities.
  - By the **content** (ontologies, clustering, keywords) → maps to relation + Output.
  - By **actionability** (PARA's Projects-first principle) → maps to Work / Output ordering.
  - By **time** (modified, created, published) → temporal ordering, often LATCH-T.
  - By **name** (alphabet, naming schemes) → LATCH-A.

## File Organization Systems (subset)

> Folded in from cyberbase's separate "File Organization Systems" page (it was 13 lines and didn't justify a standalone file).

File-organization systems differ from general information-organization systems
in that they are **constrained** by:

- **File-naming limitations** of the operating system — character set, max
  length, reserved characters, case-sensitivity rules.
- **Sorting mechanisms** of other systems that access the files — file
  explorers, web servers, sync clients each may collate differently.

These constraints flow into SEACOW(r)'s **System** component — when you're
designing for a file-system-backed knowledge platform, your structural
vocabulary has to fit through the OS's naming bottleneck.
