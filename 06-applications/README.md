## Adding a new application

1. Read [../01-framework/meta-framework.md](../01-framework/meta-framework.md) and the relevant component pages.
2. List the platform's organizational primitives (folders? tags? properties? relations? something else?).
3. For each SEACOW(r) component, note how it can be expressed in those primitives.
4. Show one or two concrete examples.
5. Call out the platform-specific constraints (what doesn't translate cleanly).
